# Launch Readiness Scorecard (Debt/Affiliate)

Score each line: **0 = not ready**, **1 = partial**, **2 = ready**

## Offer & Buyer Fit
- [ ] Offer payout and approval event confirmed in writing
- [ ] Daily cap and overflow handling defined
- [ ] Reject reasons documented
- [ ] Geo/device/time restrictions mapped

## Tracking & Routing
- [ ] Source-to-subID mapping confirmed
- [ ] Postback tested end-to-end
- [ ] Duplicate lead handling rule set
- [ ] Failed delivery fallback route set

## Funnel & UX
- [ ] Page load <3 seconds on mobile
- [ ] Form validation and error states tested
- [ ] Thank-you page + next-step messaging clear
- [ ] Call transfer flow tested (if applicable)

## Compliance
- [ ] Required disclosure text visible and legible
- [ ] Consent language for TCPA/marketing verified
- [ ] Privacy/terms pages live and linked
- [ ] State restrictions and prohibited claims removed

## Ops Control
- [ ] Day-1 KPI targets set (CPL, CR, approval rate)
- [ ] Kill criteria set before launch
- [ ] Owner assigned for hourly checks
- [ ] Incident response path defined

## Scoring
- Max score: 40
- 34–40: Launch ready
- 26–33: Launch with guardrails
- <=25: Do not launch; fix blockers first
